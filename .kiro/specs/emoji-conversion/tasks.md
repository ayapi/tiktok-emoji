# Implementation Plan

- [x] 1. プロジェクト初期化
  - pnpm でパッケージを初期化し、TypeScript strict モード、tsup によるデュアルビルド（ESM + CJS）、Vitest を設定する
  - エントリポイントを作成し、パブリック API のエクスポート構造を準備する
  - _Requirements: 3.1_

- [ ] 2. 絵文字データモジュールの構築
- [ ] 2.1 絵文字設定の型定義とデフォルトデータの作成
  - CDN ベース URL と絵文字コード配列を持つ設定型を定義する
  - 調査資料記載の全24種 CustomEmoji をデフォルト設定として定数化する
  - _Requirements: 3.1, 3.2_

- [ ] 2.2 絵文字コードから CDN URL を解決する関数の実装
  - 絵文字コード（例: `[wow]`）のブラケットを除去し、`{base_url}/{name}.webp` 形式の URL を構築する
  - 設定リストに存在しない絵文字コードの場合は undefined を返す
  - _Requirements: 2.1, 2.2_

- [ ] 3. テキスト変換エンジンの実装
  - テキスト内の `[emoji_name]` パターンを正規表現で走査し、既知の絵文字を `<img>` 要素に置換する
  - 未知の絵文字コードは元テキストをそのまま維持する
  - 絵文字コード以外のテキスト部分は変更しない
  - カスタム設定が渡された場合はデフォルト設定の代わりに使用する
  - _Requirements: 1.1, 1.2, 1.3, 2.1, 2.2, 2.3, 3.3_

- [ ] 4. コア変換ロジックのユニットテスト
  - 単一絵文字・複数絵文字・絵文字なし・未知の絵文字コードの各パターンをテストする
  - URL 解決関数の既知/未知パターンをテストする
  - カスタム設定の適用を検証する
  - _Requirements: 1.1, 1.2, 1.3, 2.1, 2.2, 2.3, 3.3_

- [ ] 5. HTTP API エンドポイントの実装
- [ ] 5.1 POST /convert ハンドラの実装
  - `node:http` で HTTP サーバーを構築し、`POST /convert` エンドポイントを提供する
  - リクエストボディの JSON パースと `text` フィールドのバリデーションを行う
  - コア変換エンジンを呼び出し、変換済み HTML を JSON レスポンスとして返す
  - オプションの `config` フィールドでカスタム設定を受け付ける
  - _Requirements: 4.1, 4.2, 3.3_

- [ ] 5.2 エラーレスポンスの実装
  - 空リクエスト・`text` フィールド欠落・不正 JSON に対して 400 エラーを返す
  - 不正な Content-Type に対して 400 エラーを返す
  - POST 以外のメソッドに対して 405 エラーを返す
  - _Requirements: 4.3, 5.1, 5.2_

- [ ] 6. 統合テストとビルド検証
  - HTTP エンドポイントに対する統合テスト（正常変換・各種エラーケース・カスタム設定）を実装する
  - tsup ビルドが ESM / CJS 両形式で正常に出力されることを確認する
  - エントリポイントからコア関数が正しくエクスポートされることを検証する
  - _Requirements: 4.1, 4.2, 4.3, 5.1, 5.2_
